# Copilot Learnings

## Purpose
This file contains project-specific learnings and conventions.

Check this file for:
- Architecture patterns
- Dependency usage
- Known limitations
- Domain terminology

## Do NOT
1. Do not force language – follow user preference
2. Do not rewrite working code without clear reason
3. Do not duplicate documentation
4. Do not guess when uncertain – ask instead
5. Do not ignore existing patterns
6. Do not delete anything from the docs-folder or README-file.

## Learnings
- Avoid creating large components → split early
- Prefer local state before introducing global state
- Always define types before implementing logic
